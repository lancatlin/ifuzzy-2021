Motivation

  • Taking pictures of slide in the class
  • Not at a good position
  • Hard to read and organize

Perspective Transform

Not very convenient

Need corner point detection

The images are complicated,

so are the algorithms

Similar background

Method

  1. User select the corner points manually
  2. Program create a RGB mapping function for the background
  3. User switch to the next image
  4. Program detect the corners for user

Remove the slide

Calc histogram

Normalize

Binarize

Apply RGB mapping function

Apply erosion and dilation

Apply connected components

Find the nearest pixel to the corners

Experiment Results

Limits

slide and background have similar color

Try other color models (HSV, YCrCb)

Conclusion

  • A simple but effective method to detect corners
  • Use histogram, erosion / dilation, connected components to filter background
  • The threshold has good fault tolerance
  • Add image adjustment features (contrast, brightness)
